## 📋 一、需求分析

### 1.1 功能需求

#### 1.1.1 项目管理

| 功能模块 | 需求描述 | 优先级 |
|:---------|:---------|:-------|
| 创建项目 | 支持创建新项目，包含项目名称、描述、创建时间 | P0 |
| 编辑项目 | 支持修改项目基本信息 | P0 |
| 删除项目 | 支持删除项目（需二次确认） | P0 |
| 项目列表 | 展示所有项目，支持按名称/状态筛选 | P0 |
| 项目详情 | 查看项目的详细信息和所有任务 | P0 |

#### 1.1.2 进度控制

| 功能模块 | 需求描述 | 优先级 |
|:---------|:---------|:-------|
| 任务创建 | 在项目中创建任务，包含标题、描述、截止日期 | P0 |
| 任务编辑 | 修改任务的所有属性 | P0 |
| 任务删除 | 删除任务（需二次确认） | P0 |
| 任务状态 | 支持任务状态流转：待处理 → 进行中 → 已完成 | P0 |
| 进度统计 | 自动计算项目完成百分比 | P1 |
| 任务排序 | 支持按状态、截止日期、创建时间排序 | P1 |

#### 1.1.3 目标管理

| 功能模块 | 需求描述 | 优先级 |
|:---------|:---------|:-------|
| 设置目标 | 为项目设置可量化的目标（如：完成20个任务） | P1 |
| 目标追踪 | 显示当前进度与目标的差距 | P1 |
| 目标达成 | 目标达成时给出可视化提示 | P2 |

#### 1.1.4 状态追踪

| 功能模块 | 需求描述 | 优先级 |
|:---------|:---------|:-------|
| 项目状态 | 支持项目状态：规划中 / 进行中 / 已完成 / 已搁置 | P0 |
| 看板视图 | 以看板形式展示任务（待处理/进行中/已完成） | P1 |
| 时间线视图 | 以时间线形式展示任务进度 | P2 |
| 数据统计 | 提供项目维度的统计数据（完成率、逾期任务等） | P1 |

#### 1.1.5 项目文件夹集成

| 功能模块 | 需求描述 | 优先级 |
|:---------|:---------|:-------|
| 打开项目文件夹 | 通过系统文件选择器选择项目所在文件夹 | P0 |
| 状态文件读写 | 在项目文件夹中创建/读取 .task-manager.json 状态文件 | P0 |
| 状态文件同步 | 自动将项目状态同步到 .task-manager.json 文件 | P1 |
| 跨设备共享 | 支持通过项目文件夹实现状态共享（文件放在项目目录中） | P1 |

### 1.2 非功能需求

#### 1.2.1 性能需求

- 页面加载时间 < 2秒
- 操作响应时间 < 500ms
- 支持至少 100 个项目同时在线

#### 1.2.2 兼容性需求

- 浏览器支持：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 响应式设计，支持 1024px 以上屏幕

#### 1.2.3 数据安全

- 用户数据本地存储
- 支持数据导出（JSON 格式）
- 支持数据备份

#### 1.2.4 可用性

- 操作错误时可撤回
- 关键操作有确认提示
- 清晰的错误提示信息

## 🛠️ 二、技术栈选择

### 2.1 前端技术栈

| 技术 | 选择 | 理由 |
|:-----|:-----|:-----|
| 框架 | React 18 | 生态成熟，组件化开发效率高 |
| 构建工具 | Vite | 开发体验好，热更新快 |
| 路由 | React Router v6 | React 官方推荐 |
| 状态管理 | Zustand | 轻量级，API 简洁 |
| UI 组件 | Tailwind CSS + Headless UI | 高度定制化，快速开发 |
| 图标 | Lucide React | 图标库丰富，设计美观 |
| 日期处理 | Day.js | 轻量级，API 与 Moment.js 兼容 |
| 文件系统 | File System Access API / Electron | 读取项目文件夹，读写 .task-manager.json |

### 2.2 后端技术栈（可选 - 考虑未来扩展）

| 技术 | 选择 | 理由 |
|:-----|:-----|:-----|
| 运行时 | Node.js 20 LTS | 成熟稳定，生态丰富 |
| 框架 | Express.js | 轻量灵活，学习成本低 |
| 数据库 | SQLite (开发) / PostgreSQL (生产) | 轻量且支持 SQL |
| ORM | Prisma | 类型安全，迁移方便 |

> **注意**：本期 MVP 先采用 localStorage 实现纯前端版本，后续可平滑迁移到后端。

### 2.3 技术栈总结

```
前端: React 18 + Vite + TypeScript + Tailwind CSS + Zustand + React Router
后端: 暂不考虑（使用 localStorage 替代）
数据库: localStorage（未来迁移到 PostgreSQL）
文件系统: File System Access API（Web）/ Electron（桌面端）
```

## 🏗️ 三、项目架构设计

```
task-manager/
├── src/
│   ├── components/          # UI 组件
│   │   ├── common/          # 通用组件（Button, Input, Modal 等）
│   │   ├── project/         # 项目相关组件
│   │   ├── task/            # 任务相关组件
│   │   └── layout/          # 布局组件
│   ├── pages/               # 页面
│   │   ├── Dashboard/       # 仪表盘
│   │   ├── ProjectList/     # 项目列表
│   │   └── ProjectDetail/   # 项目详情
│   ├── stores/              # Zustand 状态管理
│   │   ├── projectStore.ts  # 项目状态
│   │   └── taskStore.ts     # 任务状态
│   ├── hooks/               # 自定义 Hooks
│   ├── utils/               # 工具函数
│   ├── types/               # TypeScript 类型定义
│   ├── App.tsx
│   └── main.tsx
├── public/
├── index.html
├── package.json
├── vite.config.ts
├── tailwind.config.js
└── tsconfig.json
```

### 3.2 数据模型

#### Project（项目）

```typescript
interface Project {
  id: string;              // 唯一标识 (UUID)
  name: string;            // 项目名称
  description: string;    // 项目描述
  status: ProjectStatus;  // 项目状态
  goal?: ProjectGoal;     // 项目目标（可选）
  path?: string;           // 项目文件夹路径（可选，用于文件夹集成）
  createdAt: string;      // 创建时间
  updatedAt: string;      // 更新时间
}

type ProjectStatus = 'planning' | 'in_progress' | 'completed' | 'suspended';
```

#### ProjectFolder（项目文件夹配置）

```typescript
interface ProjectFolder {
  projectId: string;       // 关联的项目 ID
  folderPath: string;      // 文件夹路径 (File System Access API handle 或 path)
  stateFileName: string;   // 状态文件名，默认 .task-manager.json
  lastSyncedAt: string;    // 最后同步时间
}
```

#### Task（任务）

```typescript
interface Task {
  id: string;              // 唯一标识 (UUID)
  projectId: string;       // 所属项目 ID
  title: string;           // 任务标题
  description: string;     // 任务描述
  status: TaskStatus;      // 任务状态
  dueDate?: string;        // 截止日期（可选）
  createdAt: string;       // 创建时间
  updatedAt: string;       // 更新时间
}

type TaskStatus = 'todo' | 'in_progress' | 'done';
```

#### ProjectGoal（项目目标）

```typescript
interface ProjectGoal {
  targetCount: number;    // 目标完成数量
  currentCount: number;    // 当前完成数量
  description: string;    // 目标描述
}
```
